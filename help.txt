 Advanced RAG Techniques Used

 1. BASIC RAG
   └── Simple retrieve → Generate

2. 🔄 MULTI-QUERY RAG (We Use This)
   ┌─────────────────────────────────┐
   │ User Query                      │
   │        ↓                        │
   │ Generate Multiple Variations:   │
   │ ├── "non-compete enforceability"│
   │ ├── "employment restrictions"   │
   │ └── "post-job work limitations" │
   │        ↓                        │
   │ Retrieve for each variation     │
   │        ↓                        │
   │ Merge & deduplicate results     │
   └─────────────────────────────────┘

3. 🔗 SEMANTIC CHUNKING (We Use This)
   └── Split documents by meaning,
       not by fixed character count

4. 🏷️ HYBRID SEARCH (We Use This)
   ┌─────────────────────────────────┐
   │ Combine:                        │
   │ ├── Vector Search (semantic)    │
   │ ├── Keyword Search (exact)      │
   │ └── Metadata Filter (jurisdiction│
   │     category, date)            │
   └─────────────────────────────────┘

5. 📊 RE-RANKING (We Use This)
   ┌─────────────────────────────────┐
   │ Initial retrieval → 10 results  │
   │         ↓                       │
   │ Cross-encoder re-rank           │
   │         ↓                       │
   │ Final top-3 results to LLM      │
   └─────────────────────────────────┘

6. 🧠 CONTEXTUAL COMPRESSION (We Use This)
   └── Extract only relevant parts
       from retrieved documents
       (not entire document)
